# V1.3 发布说明：Task Center 可靠性与交付完整性

## Release Notes

V1.3 聚焦任务中心稳定性与交付质量，主要改动如下：

- 修复并加固 `public/index.html` 的 Task Center UI 结构，避免后续出现重复 `id` 不可见问题。
- 任务列表新增：
  - 排序选择（最新优先 / 最早优先）
  - 导出当前筛选结果为 JSON
  - 最近刷新时间显示
- 日志列表新增（中文优先）：
  - 筛选项：`limit`、`status`、`action/type` 关键字、`target` 关键字
  - 应用 / 重置
  - 排序选择（最新优先 / 最早优先）
  - 导出当前筛选结果为 JSON
  - 最近刷新时间显示
- 兼容性与稳健性加固：
  - 任务/日志行渲染对缺失字段、`null`、异常对象做保护，避免渲染抛错
  - 详情弹窗对畸形对象安全展示，不因异常结构崩溃
  - 自动刷新与页面生命周期清理加强，避免重复定时器与泄露
- UI 合约检查增强：
  - 增加 HTML 重复 `id` 检测（存在即失败）
  - 保留既有关键 Hook/ID 检查并补充 V1.3 新增元素

## 手工 QA 清单

1. 登录后进入「任务中心」，确认页面可正常渲染且无报错。
2. 在任务筛选中设置 `limit/type/status/target`，点击「应用筛选」，确认任务列表刷新。
3. 修改任务排序（最新/最早），确认列表顺序变化；分页仍可正常翻页。
4. 点击任意任务行，确认详情弹窗可打开并可通过按钮、遮罩、`Esc` 关闭。
5. 点击「导出当前任务(JSON)」，确认浏览器下载 JSON 文件且内容与当前筛选一致。
6. 在日志筛选中设置 `limit/status/action/type 关键字/target 关键字`，点击「应用筛选」，确认结果变化。
7. 点击日志排序切换（最新/最早），确认顺序变化。
8. 点击「导出当前日志(JSON)」，确认下载 JSON 文件且内容与当前筛选一致。
9. 点击「刷新任务 / 刷新日志」，确认对应“最近刷新”时间更新。
10. 开启任务中心自动刷新（10s/20s/30s）后等待一个周期，确认任务与日志持续刷新且无重复请求风暴。
11. 切换到其他视图或退出登录，确认自动刷新停止。
12. 用异常数据（可手工改 `data/*.json` 注入 `null`/缺字段对象）验证任务/日志行和详情弹窗不崩溃。
13. 执行验证命令并确认通过：

```bash
node --check public/app.js
npm run check:ui
npm run smoke
npm run verify
```
