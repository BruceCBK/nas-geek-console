# V1.2 Task Center UX Enhancements

## Release Note

V1.2 focuses on the Task Center experience with safer, faster monitoring:

- Added colored status badges for task/log statuses: `success`, `failed`, `running`, `pending`, `unknown` (and compatible variants).
- Added a detail modal: clicking a task row or log row opens full details (`id`, `type`, `target`, `message`, `error`, timestamps, raw JSON).
- Added Task Center auto-refresh controls with toggle + interval selector (`off`, `10s`, `20s`, `30s`).
- Added client-side pagination for task list (Prev/Next + page indicator) using fetched task results.
- Added timer lifecycle cleanup to prevent duplicate timers and memory leaks when switching view/tab or logout.
- Updated UI contract checks with new Task Center critical element IDs.

## Manual Validation Checklist

1. Open Task Center and confirm every task/log row status badge has expected color and Chinese label.
2. Click a task row and verify detail modal shows:
   - ID / Type / Target / Status / Message / Error / CreatedAt / EndedAt
   - `Raw JSON` block
3. Click a log row and verify detail modal shows:
   - ID / Action / Type / Target / Status / Message / Error / CreatedAt / EndedAt
   - `Raw JSON` block
4. Close modal using each method:
   - Close button
   - Backdrop click
   - `Esc` key
5. In Task Center, set auto-refresh interval to `10s`, enable toggle, wait one cycle, and confirm tasks/logs refresh.
6. Change interval to `20s` or `30s` while enabled and confirm only one timer remains active (no duplicate refresh burst).
7. Switch Task Center to another view and verify auto-refresh stops.
8. In content view, switch tabs and verify Task Center auto-refresh remains stopped.
9. Logout and verify Task Center auto-refresh stops without errors.
10. Set interval to `off` and confirm toggle is disabled and timer is cleared.
11. Confirm task pagination:
    - Prev/Next buttons work
    - Page indicator updates
    - Boundary button states are correct on first/last page
12. Run validation scripts:
    - `node --check public/app.js`
    - `npm run check:ui`
    - `npm run smoke`
    - `npm run verify`
