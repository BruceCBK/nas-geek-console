# V1.4 Memory Workspace Enhancements

## Feature List

- Back-end memory APIs remain compatible with existing endpoints.
- Added memory file deletion API with protection for `MEMORY.md`.
- Added memory file rename API.
- Added memory compression API with preview/apply mode:
  - trims trailing spaces
  - collapses excessive blank lines
  - removes adjacent duplicate lines conservatively
  - avoids aggressive changes inside markdown code fences
- Added memory extraction API to locate relevant snippets across memory files.
- Added operation logs for memory create/save/delete/rename/compress actions.
- Memory Workspace UI now supports:
  - 删除文件
  - 重命名文件
  - 一键压缩(预览)
  - 应用压缩
  - 搜索提取
- Added extraction results panel; clicking “追加片段” appends snippet into editor with separator block.
- Destructive actions (delete/apply compression) require confirmation.
- When no file is selected, file-scoped action buttons are disabled and warning is shown.

## API Examples

All examples assume authenticated requests with `Authorization: Bearer <token>`.

### Delete file

```bash
curl -X DELETE 'http://127.0.0.1:3900/api/memory/file?path=notes/test.md' \
  -H 'Authorization: Bearer <token>'
```

### Rename file

```bash
curl -X POST 'http://127.0.0.1:3900/api/memory/rename' \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer <token>' \
  -d '{"fromPath":"notes/old.md","toPath":"notes/new.md"}'
```

### Compress preview

```bash
curl -X POST 'http://127.0.0.1:3900/api/memory/compress' \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer <token>' \
  -d '{"path":"notes/new.md","apply":false}'
```

### Compress and apply

```bash
curl -X POST 'http://127.0.0.1:3900/api/memory/compress' \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer <token>' \
  -d '{"path":"notes/new.md","apply":true}'
```

### Extract snippets

```bash
curl 'http://127.0.0.1:3900/api/memory/extract?q=keyword&limit=10' \
  -H 'Authorization: Bearer <token>'
```

Response payload fields:

- `path`
- `lineStart`
- `lineEnd`
- `score`
- `snippet`

## Release Steps

1. Run required validations:

```bash
node --check public/app.js
npm run check:ui
npm run smoke
npm run check:memory
npm run verify
```

2. Revert runtime drift before commit:

```bash
git checkout -- data/*.json
```

3. Commit:

```bash
git add src/routes/memory.js src/services/memory-service.js public/index.html public/app.js public/styles.css scripts/ui-contract-check.js scripts/memory-workspace-check.js package.json docs/v1.4-memory-workspace.md
# if package-lock.json changed, include it as well
```

```bash
git commit -m 'feat(nas-console): enhance memory workspace with compress, extract and full CRUD'
```

4. Restart process safely and verify:

```bash
# restart command depends on runtime manager in your environment
curl -sS http://127.0.0.1:3900/api/health
```

## Rollback

- Preferred code rollback:

```bash
git revert <commit_hash>
```

- If urgent runtime rollback is needed, restart service with previous deployed revision and verify `/api/health`.
